## Design Context

### Users
Individual developers who track work-in-progress across multiple services (GitHub PRs, Azure DevOps, shell commands). They use Cortado as a glanceable menubar companion — a quick check-in tool, not a deep-dive dashboard. They value speed, low resource usage, and staying in flow.

### Brand Personality
**Modern, precise, clean.** Cortado is named after the balanced coffee drink — neither too strong nor too weak. The interface should feel calibrated and intentional, like a well-made instrument.

### Aesthetic Direction
- **Custom/branded** — not a clone of macOS System Settings, but still clean and native-feeling
- **Both light and dark mode** — follows system preference (already implemented in the panel)
- **Color system**: OKLCH-based, blue-gray neutrals (hue 250), teal accent (hue 178), semantic status colors (red/yellow/blue/green)
- **Typography**: SF Pro Text / system fonts for panel; Space Grotesk for settings window
- **Panel surface**: Native macOS vibrancy/blur, popover chrome
- **Settings window**: Standard decorated window, L5b layout (sidebar nav + two-line cards), F2 breadcrumb replace for edit flows
- **No UI component library** — custom CSS throughout

### Design Principles
1. **Precision over decoration** — Every element earns its place. No filler, no chrome for chrome's sake.
2. **Native-adjacent, not native-clone** — Feel at home on macOS without being a System Settings impersonator.
3. **Glanceable hierarchy** — Information density is high but never overwhelming. Status is always clear at a glance.
4. **Responsive restraint** — Animations are purposeful and fast. Motion serves comprehension, not spectacle.
5. **Developer-respectful** — The UI trusts the user's intelligence. Progressive disclosure over hand-holding.
