Design Brief — Split-Panel Drilldown

Five sub-variants of the Spotlight + Split concept. All share: floating centered overlay, global hotkey trigger, ↑↓ to navigate the activity list, detail pane updates live with focus, Enter opens the activity URL, Esc closes. The variants explore differences in: filter bar presence, list density, detail placement, feed identification method, and the "needs attention" cross-feed section idea.

Fidelity Notes
  • Active/focused row shown with blue outline. In all variants, arrow-key focus drives the detail pane content.
  • Data matches FeedSnapshot/Activity/Field types. Three feeds: GitHub PRs (3 activities), ADO PRs (2), Deploy Status (2, 1 retained).
  • Filter bar is passive in all variants — panel opens showing full list, typing activates filter.

1 · Clean Split

Full-width filter bar at top. Left list shows dot + title only (no chips) to keep it narrow. Right detail pane with feed label, title, fields, and open link. Classic balanced proportions — 220px list, ~340px detail.

Trade-off: The list is minimal — you need to look at the detail pane to see status text. Good if the dot color is enough for scanning.

Filter bar Minimal list Balanced split Feed headers in list

2 · Borderless

No filter bar — the panel opens directly into the content. List shows dot + title + inline status chip. Detail pane has a subtle tinted background and a larger title with the status dot beside it. Calmer and more spacious.

Trade-off: No filter means you always scroll/arrow to find things. But with 5-15 activities, that's fast. The inline chips mean the list is wider.

No filter Inline chips Tinted detail pane Spacious

3 · Compact

Tightest variant. List uses color-coded left-edge bars instead of feed section headers — saves vertical space. Filter bar is slim. Detail pane shows status chips in a row, then fields. Feels fast and dense.

Trade-off: Color bars need a legend (or learning). Row height is minimal, requiring precise arrow navigation. Best for power users comfortable with density.

Color-coded feeds No feed headers Dense rows Slim filter

4 · Stacked

Vertical split — list on top, detail below — in a narrower panel (400px). Closer to the current menubar panel's single-column feel. Detail pane uses a 2-column field grid to stay compact. Familiar mental model.

Trade-off: Less horizontal space means shorter titles truncate earlier. The detail pane competes with the list for vertical space — shorter list visible at once.

Vertical split Narrow (400px) 2-col fields Menubar-like

5 · Priority + Split

Widest variant. List starts with a "Needs Attention" cross-feed section (attention-negative items from any feed), followed by per-feed groups. Items in the attention section show a feed hint label. Detail pane includes a styled "Open" button instead of a text link. The hybrid of A+C+D ideas.

Trade-off: The attention section duplicates items that also appear in their feed group — or do we remove them from the feed group? Either way, it adds UX complexity. The wider size (600px) feels more like a window than an overlay.

Cross-feed attention section Feed groups below Styled CTA button Widest (600px)
