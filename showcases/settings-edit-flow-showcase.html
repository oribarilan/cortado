Feed Edit Flow — Patterns

Four ways to handle "Add feed" / "Edit feed". Same L5b layout with teal accent, same form fields — different interaction patterns.

E1 — Inline Expansion

In-place form

The form appears inline, expanding below the "Add feed" button (or replacing the feed card being edited). No navigation — the user stays in context. Compact, but the form is constrained to the feed list width.

E2 — Slide-over Panel

Side panel overlay

A panel slides in from the right, overlaying part of the feed list. The feed list stays visible (dimmed) for context. The form gets its own dedicated vertical space. Close button or "Cancel" dismisses.

E3 — Replace Content

Full-area form

The main area replaces the feed list entirely with the form. A back arrow returns to the list. Maximum form space — no competition with the list. Feels like navigating deeper into a detail view.

E4 — Modal Dialog

Centered overlay

A modal dialog overlays the settings window. The feed list stays visible but inert behind it. Focused attention on the form. Common pattern for "create/edit" flows. Escape or clicking outside dismisses.
