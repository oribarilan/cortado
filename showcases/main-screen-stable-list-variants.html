Stable List + Detail Variants

The single-pane expandable layout causes jumpiness — the list shifts as detail opens/closes. These variants bring back a separate detail area so the list stays stable, while giving titles more room than the original 280px split.

Core constraint: The activity list must not shift or jump when focus changes. Detail should live in a dedicated area — right pane, bottom drawer, or narrow sidebar. The list should be wider than the original 280px and show inline status chips for at-a-glance triage.

F1

Flex split — wide list (60%) + inset detail pane

The list takes 60% of the window with inline status chips. The detail pane takes 40% with an inset background to visually separate it. Full title shown in the detail. List never moves.

F2

Full-width list + bottom detail drawer

The list takes the full width (maximum title space). A fixed-height drawer at the bottom shows the focused item's detail. The list is completely stable — the drawer is always the same height. Fields are laid out horizontally to fit the wide format.

F3

Wide list + narrow detail sidebar

The list gets ~400px (most of the window). A narrow 160px sidebar on the right shows the focused item's fields stacked vertically with labels above values. Maximum list space with a persistent detail glance.

F4

60/40 split — no detail chrome, just title + fields

Like F1 but the detail pane has zero chrome — no feed label, no status chip, no inset background. Just the full title (separated by a rule) and the raw field list. The lightest possible detail pane — maximum list width, minimum detail overhead.
