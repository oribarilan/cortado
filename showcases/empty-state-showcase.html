★ Recommendation: C2
C2 (Main Screen + Split-Pane Creation) is the strongest option. Here's why:

Reuses existing patterns. The main screen already has a split pane with list + detail. C2 repurposes that layout — feed types in the list pane, form in the detail pane — so it feels native to the app from the very first interaction. No modals, no separate windows, no new UI primitives.

Room to breathe. At 840 x 480, the main screen has space for both feed type discovery and a comfortable form. Compare this to C1, where a modal inside a 350px panel is cramped, or B1/B2, which require a separate window the user won't see again.

Zero context switches. The user starts in the main screen, creates a feed there, and the screen instantly shows the feed — no wizard-to-panel handoff, no "Open Cortado" button, no window juggling. The FRE ends naturally by becoming the normal app.

Teaches the right UI first. The main screen is where users will spend their time. By opening it on first launch (instead of the panel), users immediately learn the app's primary surface. The panel is the "quick glance" secondary — they'll discover it via the hotkey hint.

Escape hatch built in. Power users can dismiss the creation flow, click the gear icon to open Settings, or edit the TOML directly. The secondary "or edit feeds.toml" link respects developer intelligence without cluttering the primary flow.
