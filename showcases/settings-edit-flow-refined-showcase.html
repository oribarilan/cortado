Edit Flow — Refined Alternatives

Three patterns I'd recommend. Each shown in two states: the feed list view (left) and the editing state (right). All use L5b + teal, with the config path bar and "Open in editor" / "Reveal in Finder" actions.

F1 — Sidebar Drill-Down

Recommended

Clicking a feed adds it as a sub-item under "Feeds" in the sidebar. The main area shows the form. The user always knows where they are — the sidebar shows the navigation path. Click "Feeds" to go back. Feels like a native macOS settings pattern (System Settings uses this exact model).

F2 — Breadcrumb Replace

Clean navigation

Like E3 (replace content) but with a breadcrumb instead of a bare back arrow. "Feeds › My PRs" makes the navigation path explicit. The sidebar stays unchanged — "Feeds" remains the active nav item. Clicking "Feeds" in the breadcrumb returns to the list.

F3 — List-Detail Split

Always visible list

When editing, the content area splits: a narrow feed list stays on the left (showing names only), and the form fills the right. The selected feed is highlighted. You can switch between feeds without going "back" first. The window is wider (780px) to accommodate the split. Like a mail client: list left, detail right.
