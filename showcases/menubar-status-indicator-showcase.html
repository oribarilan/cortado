Menubar Icon — Status Indicator Alternatives

The tray icon should passively signal the global rollup status (highest severity across all feeds). This showcase compares five approaches to rendering that indicator, in both dark and light menubar themes.

A: Colored Dot Overlay

Recommended

A small colored circle composited onto the bottom-right corner of the icon at runtime. Like Teams or Slack status dots. Requires disabling template mode and handling theme manually.

Pros

  • Dynamic — 1 base asset, computed colors
  • Matches Teams/Slack pattern
  • Compact — no extra menubar space

Cons

  • Must handle light/dark theme manually
  • Animation limited without rapid swaps
  • Dot small at native 22px

C: Separate Dot Item

Low effort

A second, independent tray item — a tiny colored circle — appears next to the main icon. Base icon stays as template (auto light/dark). Simplest code path.

  • Looks disconnected from icon
  • Unusual pattern — may confuse users

D: Icon Shape Swap

HIG-compliant

The entire icon changes to indicate status — e.g., filled glass for attention, half-filled for waiting, hollow outline for idle. Stays monochrome template. Respects macOS HIG strictly.

Pros

  • Full template mode — auto theme
  • Strictly HIG-compliant
  • No color in menubar

Cons

  • Monochrome — less glanceable
  • Hard to design 5 distinct silhouettes at 22px
  • Users must learn icon vocabulary

E: Native Subview (NSStatusItem)

High effort

A native macOS NSView added as a subview on the tray button via objc interop. Supports smooth animations (pulse for Running). Best quality but most complex and fragile.
