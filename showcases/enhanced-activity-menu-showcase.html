Fidelity Notes — HTML vs Native Menubar Components
  • Vibrancy: real NSMenu/NSPanel vibrancy compositing cannot be exactly reproduced in HTML.
  • Font precision: SF Pro optical weights vary by environment; browser fallback may shift weight/spacing.
  • Colored row accents: per-item borders/backgrounds are not available in native NSMenu APIs (exact limitation being explored).
  • Submenu hit behavior: NSMenu has system hover-delay triangles for submenus; this mock is direct hover/click.
  • Scroll feel: NSScrollView rubber-banding and inertia differ from browser overflow scrolling.
  • Retained visuals: dashed retained styles are design proposals; native menu equivalent today is the hollow-dot symbol.

Variant Notes & Tradeoffs

A — Color-Banded Rows

Native-like list rows with a thin severity band on the left edge. Feed labels stay full-opacity and readable while Activity rows surface severity with minimal visual noise.

B — Control-Center Cards

Activity cards use subtle severity-tinted backgrounds and inline Field chips, inspired by Control Center and Wi-Fi module styling while keeping compact menubar density.

C — Traffic-Light Sidebar

A dedicated gutter carries severity color for each Activity. The content remains mostly neutral text, so color coding is strong without overwhelming typography.

D — Inline Severity Stripe + Expandable Detail

Closest to current submenu mental model: compact Activity rows with expand/collapse details that reveal all Fields inline, with color-coded status values.
