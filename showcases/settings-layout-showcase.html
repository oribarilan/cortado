Layout Comparison

Same tokens, same accent, same typography. Four layouts that differ only in navigation pattern and feed presentation. Accent/theme is a separate choice.

L1 — Sidebar + Table

Two-Pane / Tabular

Classic settings layout. Sidebar for section switching, table for feed data. Most scannable for many feeds — columns give instant comparability. Sidebar takes ~176px of horizontal space.

L2 — Sidebar + Flat List

Two-Pane / Minimal

Same two-pane as L1 but feeds are a flat list instead of a table. Less chrome, more breathing room. Hover reveals a navigation arrow. Works well for fewer feeds where comparability across columns isn't needed.

L3 — Segmented Nav + Cards

Top Bar / Cards

No sidebar — navigation via a segmented pill control in a top bar. Feeds as compact cards with left-edge indicators and hover-revealed actions. Full content width. Most "app-like" feel — each feed gets its own visual container.

L4 — Single Column / Sections

No Nav / Document

No sidebar, no tabs. A single scrollable column with collapsible section headers. Feeds as a compact inline list with status dots. Maximizes content width. Reads like a config document — closest in spirit to how the TOML file itself is structured.
