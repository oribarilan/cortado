Fidelity Gaps — HTML vs Native NSMenu
  • Vibrancy: NSMenu uses system vibrancy material; HTML backdrop-filter approximates but cannot replicate the real compositing.
  • Font rendering: macOS renders SF Pro with subpixel hinting in menus; browser rendering differs slightly in weight/spacing.
  • Submenu behavior: Real NSMenu submenus slide in with a native delay zone (triangular tolerance); hover-show here is instant and axis-aligned.
  • NativeIcon: StatusAvailable / StatusUnavailable are system-provided SF Symbols; we simulate with colored circles.
  • Row height: Native rows are 22px with precise vertical centering; browser line-height may differ by ±1px.
  • Separator rendering: NSMenu separators are 1px with system separatorColor; HTML uses a close match.
  • Menu positioning: Real tray menus anchor to the menubar with a pointer; this showcase simulates standalone menus.
  • Keyboard nav: NSMenu supports arrow-key navigation natively; this showcase is visual-only.
