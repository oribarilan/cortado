Design Brief — Premium Menubar Dropdown

Problem 1: Feed headings rendered as disabled MenuItem in NSMenu look grayed-out and "broken" — users think they're supposed to be clickable. Problem 2: Activity items are flat submenus with text-only status symbols — no color coding is visible at the top-level menu, making it impossible to glance and know what needs attention.

These 4 variants explore how a custom panel (NSPanel via Tauri webview, replacing the native NSMenu) solves both problems while feeling like a premium macOS dropdown — inspired by Control Center, Wi-Fi, and Bluetooth menus.

Fidelity Gaps vs Native
  • Vibrancy: NSVisualEffectView material cannot be replicated — backdrop-filter approximates.
  • Font rendering: SF Pro optical weight/subpixel hinting differs between native and browser.
  • Scroll feel: NSScrollView rubber-banding differs from browser overflow scroll.
  • Shadow + border: System menu shadow has adaptive ambient occlusion; CSS box-shadow is static.
