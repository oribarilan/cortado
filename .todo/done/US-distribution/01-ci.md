# CI Pipeline

## Goal

Set up a GitHub Actions CI workflow that runs on every push and PR to validate code quality and build integrity.

## Acceptance criteria

- [x] GitHub Actions workflow at `.github/workflows/ci.yml`
- [x] Runs on macOS ARM runner (`macos-latest`)
- [x] Steps: install Rust + Node.js + pnpm + just, then run `just check` (format + lint + test)
- [x] Build verification: `just build` (ensures the app compiles and bundles)
- [x] Workflow triggers on push to `main` and on pull requests
- [x] CI process documented in `specs/ci.md`
- [x] CI badge added to `README.md`

## Notes

- The Nix flake already provides the full dev environment. Consider using [DeterminateSystems/nix-installer-action](https://github.com/DeterminateSystems/nix-installer-action) + `nix develop` in CI for reproducibility. Alternatively, set up Rust + Node manually for faster CI if Nix is too slow.
- `just check` already runs format, lint (tsc + clippy with `-D warnings`), and test. CI should mirror local dev exactly.
- Build step is heavier -- consider whether to run it on every PR or only on pushes to `main`.
- The `cargo-clippy` feature flag workaround in Cargo.toml may need attention in CI.
