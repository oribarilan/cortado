# CD Pipeline

## Goal

Set up a GitHub Actions CD workflow that builds, signs, and publishes release artifacts to GitHub Releases when a version tag is pushed.

## Acceptance criteria

- [ ] GitHub Actions workflow at `.github/workflows/cd.yml`
- [ ] Triggers on push of `v*` tags (e.g., `v0.3.0`)
- [ ] Builds the Tauri app for macOS aarch64 (Apple Silicon)
- [ ] Signs updater artifacts with Tauri signing key (private key from `TAURI_SIGNING_PRIVATE_KEY` secret)
- [ ] Generates `createUpdaterArtifacts: true` outputs: `.app.tar.gz` + `.app.tar.gz.sig`
- [ ] Generates `latest.json` with version, platform URL, signature, and release notes from `CHANGELOG.md`
- [ ] Creates a GitHub Release with:
  - Release title: `vX.Y.Z`
  - Body: extracted from `CHANGELOG.md` for this version
  - Assets: `.app.tar.gz`, `.app.tar.gz.sig`, `latest.json`
- [ ] CD process documented in `specs/cd.md`

## Notes

- Tauri updater signing keypair must be generated first: `pnpm exec tauri signer generate -w ~/.tauri/cortado.key`. Store the private key as a GitHub Actions secret.
- The `latest.json` format for static GitHub Releases:
  ```json
  {
    "version": "X.Y.Z",
    "notes": "Release notes from CHANGELOG",
    "pub_date": "2026-03-30T00:00:00Z",
    "platforms": {
      "darwin-aarch64": {
        "url": "https://github.com/oribarilan/cortado/releases/download/vX.Y.Z/Cortado.app.tar.gz",
        "signature": "<content of .sig file>"
      }
    }
  }
  ```
- The updater endpoint in `tauri.conf.json` will point to:
  `https://github.com/oribarilan/cortado/releases/latest/download/latest.json`
- Consider using the `tauri-apps/tauri-action` GitHub Action which handles building + artifact generation.
- Extract release notes from CHANGELOG.md: parse the section between the current version header and the previous version header.
