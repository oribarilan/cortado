# CD Pipeline

## Goal

Set up a GitHub Actions CD workflow that builds a DMG and publishes it to GitHub Releases when a version tag is pushed.

## Acceptance criteria

- [ ] GitHub Actions workflow at `.github/workflows/cd.yml`
- [ ] Triggers on push of `v*` tags (e.g., `v0.3.0`)
- [ ] Builds the Tauri app for macOS aarch64 (Apple Silicon)
- [ ] Produces a `.dmg` artifact
- [ ] Creates a GitHub Release with:
  - Release title: `vX.Y.Z`
  - Body: extracted from `CHANGELOG.md` for this version
  - Assets: `.dmg`
- [ ] CD process documented in `specs/cd.md`

## Notes

- No Tauri updater signing or `latest.json` in this phase — just a DMG download. Updater artifacts are deferred to US-distribution-part2.
- Tauri builds a DMG by default on macOS when `targets` includes `"dmg"`.
- Consider using the `tauri-apps/tauri-action` GitHub Action which handles building + artifact generation.
- Extract release notes from CHANGELOG.md: parse the section between the current version header and the previous version header.
